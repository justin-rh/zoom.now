<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Railway dashboard → process.env | Credentials entered in Railway Variables tab are injected at runtime; anyone with Railway project access can read them |
| Zoom Marketplace → webhook endpoint | Zoom sends URL validation challenge to the public Railway URL; endpoint must be live before clicking Validate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | Railway environment variables | mitigate | Set ZOOM_WEBHOOK_SECRET_TOKEN and all credentials in Railway Variables tab only — never in railway.toml, never in source code; restrict Railway project access to authorized users |
| T-02-02 | Spoofing | Zoom Marketplace URL validation | mitigate | Server is already verifying HMAC signature on all POSTs (including the validation challenge) — Plan 01 Task 2 implements this; the validation challenge itself is Zoom proving the endpoint is reachable |
| T-02-03 | Denial of Service | Railway cold-start | mitigate | railway.toml has restartPolicyType = "ALWAYS" (set in Plan 01 Task 2); healthcheckPath = "/health" keeps Railway aware of liveness |
| T-02-04 | Information Disclosure | Secret Token exposure | accept | ZOOM_WEBHOOK_SECRET_TOKEN is displayed once in Zoom Marketplace UI during setup; note it immediately and store only in Railway Variables; low residual risk since it's not transmitted in webhook headers (it's used to verify them) |
</threat_model>

**STEP B: Deploy to Railway**

1. Go to https://railway.app → Log in
2. New Project → Deploy from GitHub Repo → Select the zoom.now repository
3. Railway auto-detects Node.js from package.json and builds with RAILPACK
4. Wait for the first deployment to complete (green checkmark in Railway dashboard)
5. Under Settings → Networking → Generate Domain — note the public URL (format: `https://zoom-now-production-xxxx.up.railway.app`)

**Set Railway Environment Variables** (Service → Variables tab):
```
ZOOM_WEBHOOK_SECRET_TOKEN=   ← LEAVE EMPTY for now — obtain from Zoom in Step C
ZOOM_CLIENT_ID=              ← obtain from Zoom in Step C
ZOOM_CLIENT_SECRET=          ← obtain from Zoom in Step C
ZOOM_BOT_JID=                ← obtain from Zoom in Step C
SNOW_DOMAIN=masterelectronics.service-now.com
TICKET_PREFIXES=INC,RITM,REQ
DEDUP_TTL_MINUTES=60
OPT_OUT_KEYWORD=nobot
```

6. After setting variables, Railway redeploys automatically. Wait for deployment to complete.

**Verify Railway is live:**
```bash
curl https://<your-railway-domain>.up.railway.app/health
```
Expected response: `{"status":"ok","ts":<number>}`

DO NOT proceed to Step C until this curl returns 200. The server MUST be live before clicking Validate in Zoom Marketplace.

**STEP C: Register Zoom General App in Marketplace**

Go to https://marketplace.zoom.us → Sign in with the Master Electronics Zoom admin account.

1. **Create the app:**
   - Develop menu → Build App
   - Select "General App" → Create
   - App Name: "zoom.now" (or "Master Electronics ServiceNow Bot")
   - Select how the app is managed: "Admin-managed" → Save

2. **Get credentials** (Credentials tab):
   - Copy Client ID → set as `ZOOM_CLIENT_ID` in Railway Variables
   - Copy Client Secret → set as `ZOOM_CLIENT_SECRET` in Railway Variables

3. **Enable Team Chat surface:**
   - Features → Surface → Team Chat → toggle ON
   - Bot Name: set a friendly name (e.g., "ServiceNow Bot")
   - Enable "Admin Level OAuth" → toggle ON (required for imchat:bot scope)
   - Bot JID: copy the value shown (format: `abc123@xmpp.zoom.us`) → set as `ZOOM_BOT_JID` in Railway Variables
   - Bot Endpoint URL: `https://<your-railway-domain>.up.railway.app/webhook`
   - Save

4. **Add Event Subscription:**
   - Features → Access → General Features → Event Subscription → Add New Event Subscription
   - Event notification endpoint URL: `https://<your-railway-domain>.up.railway.app/webhook`
   - Add Events → Team Chat → Chat Message → Chat Message Sent (`chat_message.sent`) → Done
   - Save

5. **Note the Secret Token:**
   - After saving, find the "Secret Token" value shown in the Event Subscription section
   - Copy this value → set as `ZOOM_WEBHOOK_SECRET_TOKEN` in Railway Variables
   - Wait for Railway to redeploy with the secret token set (takes ~30-60 seconds)

6. **Validate the endpoint:**
   - Verify Railway has redeployed with ZOOM_WEBHOOK_SECRET_TOKEN set:
     ```bash
     curl https://<your-railway-domain>.up.railway.app/health
     ```
   - Click "Validate" next to the event subscription URL in Zoom Marketplace
   - Expected: Marketplace shows "Validated" next to the URL within ~5 seconds
   - Check Railway logs for: `[webhook] URL validation challenge responded`

7. **Add OAuth scopes:**
   - Scopes section → Add Scopes:
     - `imchat:bot` (should appear automatically after Team Chat enabled)
     - `chat_message:read:admin`
   - Save

8. **Install the app (Local Test):**
   - Local Test section (or OAuth section) → Add to Zoom account
   - Complete the OAuth consent flow as admin

- `imchat:bot` scope missing → Team Chat surface not enabled or Admin Level OAuth not toggled on
- No `chat_message.sent` events received → Advanced Chat Encryption may be enabled on Master Electronics account (check with Zoom admin)
- `chat_message.sent` event type not available in event picker → Enable Team Chat surface first, then add event subscription
